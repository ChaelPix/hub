For my robot, I have to understand Inverse Kinematics to grab objects in a 3d space.

So I developed with C++ an algorithm to solve the inverse kinematics problem for a chain of segments using a geometric approach based on circle intersections. This algorithm allows the robot to accurately position its end effector at a desired location within its workspace.

The algorithm solves the inverse kinematics problem for a chain of segments using a geometric approach based on circle intersections:

  • Distance and Validity Check: The end effector's position is checked to ensure it lies within the chain's maximum reach. If it exceeds the limit, the target is scaled to the maximum reachable distance.
  • Intersection of Circles: Each segment is modeled as a circle with its radius equal to the segment length. The algorithm computes intersection points between these circles by:
    • Using the law of cosines to determine the overlap distance.
    • Calculating a perpendicular vector for the height of the triangle formed by the centers.
  • Triangle Validation: A helper function ensures that the lengths of the triangle sides formed by the segment being solved and its neighbors respect the triangle inequality, ensuring physically valid configurations.
  • Iterative Backward Resolution: Starting from the end effector, the algorithm calculates each joint's position backward toward the base by resolving circle intersections.
  • Pole Constraint: The algorithm stabilizes the chain by choosing the intersection point closer to a pole, which acts as an external constraint. This ensures smoother and predictable movement.
  • Real-Time Interaction: The algorithm dynamically updates the chain configuration in response to user input, such as moving the end effector or adjusting the pole position with the mouse.
