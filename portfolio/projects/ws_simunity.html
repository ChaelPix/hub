I set up a simulation on Unity using my Ros2 Stack to help with my robot preparation by using ros2-for-unity.

URDF loaded on Unity 3D Scene

► I translated our robot URDF on Unity with URDF Importer. Therefore I could have the robot Transforms and easily edit our robot. Then I use Unity Physics to create a /cmd_vel topic to move the robot.

Visible Tfs on RVIZ2

► Then I create a simulated LIDAR matching our YdLidar x2 with same specs and noise that publishes on the /scan topic as a real Lidar.

Visible Lidar raycasts on Unity

► The advantadge is that we can debug each raycast in order to have a better understanding. For example, a module asked us to create an algorithm from scratch in C++. I could use the simulation to debug every steps of the process

Custom Algorithm where robot move to the most empty front space

► Now with /scan, I can get the data from Unity's Scene to use it with my ROS2 Nav Stack and publishes the result on /cmd_vel.

Robot moving with SLAM

Robot moving with Nav2. (I got some fun with the scenary).

► I also link a camera with a /color/rgb/image_raw topic to use my YOLO stack for object detection.

► I started to integrate our Arm system to the Simulation.

A position is published to /point, the Inverse Kinematics Stack does the maths for arm link positions, then publishes the joints.
