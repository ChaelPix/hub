I started this project after seeing complaints online about a chatbot website that struggles with long message memory. I assumed it was due to token limits, so I challenged myself to create a solution that remembers the most important details of a conversation despite these limitations.

Selecting and chatting with a Fake AI User

Each AI user is designed with a unique personality and traits. For example, 'Ardelia' refuses to chat because she is busy working.


I also added the detection of picture request using REGEX.

The biggest challenge, beyond building the messaging system, was creating logic for picture requests while keeping response times low for both the message and the image generation.
